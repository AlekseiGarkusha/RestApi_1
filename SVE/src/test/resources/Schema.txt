### Стек - Java + Selenium + testNG

### Запуск тестов:

    0. mvn clean - Обязательный параметр перед запуском.

    1. Тесты можно запускать отдельно, для этого нужно перейти в главный класс
    (Например RegistrationClassic) ПКМ -> Run

    2. Можно запускать все тесты. Нужно нажать ПКМ на файл testng.xml --> RUN

### Перечень тестов

    ## AdminOO:

     1. AddEntranceExams - Создание вступительных испытаний.

     2. AddScoreEntranceExams - Назначение баллов за ВИ.

     3. AddGroupEntranceExams - Создание групп ВИ.

     4. CreateRoleCOO - Создание пользователей с ролью «СОО».

     5. EditingUser - Редактирование карточки пользователя.
      Проверки:
       * 1 - Изменения полей,
       * 2 - Обязательности полей,
       * 3 - Всплывающего popUp,
       * 4 - Соответствие данных

    6. BlockingUnblockingUser - Блокировка и разблокировка пользователя.

    7. ChangeApplicantData - Изменения данных заявителя в карточке
     Проверки
       * 1 - Изменение данных в карточке заявителя
       * 2 - Соответствие данных
       * 2 - Дата обновления
       * 3 - Всптывающий pop-Up


    ## CheckDuplicate

      1. CheckDuplicateAddEntranceExam - Проверка на дублирование вступительных испытаний.

      2. CheckDuplicateSpecialities - Проверка наличия дублей при создании специальностей.

      3. CheckDuplicateStatement - Валидация системы на предотвращение повторной регистрации по данным паспорта и СНИЛС.

    ##Registration

     1. RegistrationClassic
      Полная автоматизация процесса регистрации заявления (все 8 шагов формы — от авторизации до отправки заявления).

     2. RegistrationComparisonFields
      Сравнение данных, введённых при регистрации, с данными в карточке заявления.

     3. RegistrationMandatoryFields
      Проверка обязательности полей: после нажатия кнопки «Далее»
      фиксируются динамические подсказки о незаполненных обязательных полях.

### Особенности:

    1. Тесты (RegistrationClassic, RegistrationComparisonFields ) используются функции генерации рандомных:
     СНИЛС, Серии документа, Номера документа.
     Эти значения генерируются рандомно, что бы обходить проверку на дубликат заявления

    2. В классе Setup можно раскоментировать строку
     // options.addArguments("--headless");
      и проводить тесты без визуального запуска Хром,

    3. Полный цикл от подачи до статуса "Включён в приказ" автоматизировать не получится.
     т.к. нужно вручную разработчику изменять даты специальности на прошлые, что бы осуществить решение.
